# lsfMonitor v3 测试用例清单（V1.0）

> 说明：以下为首批核心用例（MVP 回归集），后续按模块扩展。

## 1. 采集链路（collect）

### TC-COL-001 采集任务正常落库
- 前置条件：LSF 命令可用、数据库目录可写
- 步骤：执行 `collect` 指令（job/queue/host/load）
- 预期：对应表新增记录，时间戳正确

### TC-COL-002 LSF 命令失败重试
- 前置条件：模拟 bjobs/bhosts 命令失败
- 步骤：执行采集
- 预期：触发重试与错误日志，不导致进程崩溃

### TC-COL-003 多进程采样完整性
- 前置条件：开启多采样项并发
- 步骤：执行一次全量采样
- 预期：所有采样子任务完成（均被 join），数据完整

### TC-COL-004 数据目录权限异常
- 前置条件：数据库目录只读
- 步骤：执行采集
- 预期：报错信息明确，主程序可控退出

## 2. 分析链路（analyze）

### TC-ANA-001 指标计算正确性
- 步骤：输入固定样本数据，执行指标计算
- 预期：成功率、浪费率、利用率与人工计算一致

### TC-ANA-002 空数据处理
- 步骤：在指定时间范围内无数据时执行分析
- 预期：返回空结果提示，不抛异常

### TC-ANA-003 大时间范围查询
- 步骤：查询 90 天趋势
- 预期：结果正确，响应在可接受范围内

## 3. 工程师 CLI

### TC-CLI-E-001 查询个人作业
- 步骤：`lsfmon my jobs`
- 预期：展示当前用户作业列表，字段完整

### TC-CLI-E-002 查询个人 memory 趋势
- 步骤：`lsfmon my mem --days 7`
- 预期：返回 7 天统计并标注异常区间

### TC-CLI-E-003 参数建议
- 步骤：`lsfmon advise --job <jobid>`
- 预期：输出建议 rusage 区间与置信等级

### TC-CLI-E-004 非法参数
- 步骤：输入非法参数/时间范围
- 预期：报错可读，返回码非 0

## 4. 管理员 CLI

### TC-CLI-M-001 总览查询
- 步骤：`lsfmon mgmt overview --range 7d`
- 预期：输出队列/主机/用户/项目核心指标

### TC-CLI-M-002 趋势报表导出
- 步骤：`lsfmon report weekly --export csv,md`
- 预期：导出文件生成成功且内容完整

### TC-CLI-M-003 过滤维度组合
- 步骤：按队列+项目+时间联合过滤
- 预期：结果与筛选条件一致

## 5. 管理员 GUI（本地）

### TC-GUI-001 页面加载
- 步骤：启动 GUI
- 预期：总览页正常加载，无阻塞崩溃

### TC-GUI-002 页签切换稳定性
- 步骤：在总览/队列/主机/用户/项目页签间反复切换
- 预期：无卡死、无异常弹窗、内存占用无明显失控增长

### TC-GUI-003 筛选联动
- 步骤：切换时间范围和队列过滤
- 预期：图表和表格同步刷新

### TC-GUI-004 图表与表格一致性
- 步骤：固定筛选条件下对比图表数值与表格聚合数值
- 预期：关键指标一致（允许可配置的四舍五入误差）

### TC-GUI-005 空数据场景
- 步骤：选择无数据时间段
- 预期：显示“无数据”提示，不崩溃，不出现脏数据

### TC-GUI-006 导出功能
- 步骤：导出 PNG + CSV +（若支持）PDF
- 预期：文件可打开，内容与当前视图一致，文件名包含时间信息

### TC-GUI-007 异常反馈可读性
- 步骤：模拟后端命令失败/数据库不可写
- 预期：GUI 提示清晰（错误原因+建议动作），日志可追踪

### TC-GUI-008 长稳运行
- 步骤：GUI 连续运行 4~8 小时并定期刷新
- 预期：无崩溃，核心功能可持续使用

## 6. 兼容性

### TC-COMP-001 CentOS 7.9 全链路
- 步骤：部署并执行 collect/analyze/CLI/GUI
- 预期：核心流程全部可用

### TC-COMP-002 Rocky 8.10 全链路
- 步骤：同上
- 预期：核心流程全部可用
