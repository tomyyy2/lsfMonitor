# lsfMonitor v3 测试策略（V1.0）

## 1. 测试目标
1. 保证 CentOS 7.9 主环境可稳定运行。
2. 保证 Rocky 8.10 兼容。
3. 保证工程师 CLI 与管理员（CLI + GUI）核心流程正确可用。
4. 保证采集与分析链路正确、可回归、可追溯。

## 2. 测试范围
- In Scope：
  - 数据采集（collect）
  - 数据分析（analyze）
  - 工程师 CLI
  - 管理员 CLI
  - 管理员 GUI（本地）
  - 报表导出
- Out of Scope（本期）：
  - Web 交互测试
  - 大规模分布式性能压测

## 3. 测试分层
1. **单元测试（Unit）**
   - 数据解析
   - 指标计算
   - 参数建议逻辑
   - SQL 读写封装
2. **集成测试（Integration）**
   - 采集→落库→分析→输出全链路
   - CLI 与数据层联调
3. **系统测试（System/E2E）**
   - 工程师核心场景
   - 管理员分析与报表场景
4. **GUI 专项测试（Desktop GUI）**
   - 启动与稳定性（首次加载、长时间运行）
   - 控件交互（筛选、页签切换、排序、联动刷新）
   - 图表与表格一致性（同一筛选条件下数据一致）
   - 导出正确性（PNG/CSV/PDF 内容一致）
   - 异常反馈（空数据、命令失败、权限不足）
5. **兼容性测试**
   - CentOS 7.9
   - Rocky 8.10

## 4. 测试框架与方法（建议落地）

## 4.1 框架选型
- `pytest`：测试主框架
- `pytest-cov`：覆盖率统计
- `pytest-mock` / `unittest.mock`：外部命令与环境 mock
- `tox`（可选）：多 Python 版本/环境执行
- GUI：采用“分层测试”策略（界面逻辑单测 + 冒烟自动化 + 人工探索测试）
  - 自动化建议：`pytest-qt`（PyQt 事件/控件交互）
  - 截图对比（可选）：关键页面快照基线对比
  - 人工测试：复杂交互、图表可读性、导出正确性

## 4.2 方法
- **等价类 + 边界值**：参数、时间范围、空数据/极值数据
- **故障注入**：LSF 命令失败、DB 锁冲突、文件权限不足
- **回归用例库**：每次修复缺陷必须补回归用例
- **可复现基线数据**：构造固定样本库，保证分析结果可对比

## 5. 质量门禁（建议）
- 单元测试通过率：100%
- 核心模块覆盖率：>= 70%（逐步提升到 80%）
- P0/P1 缺陷：发布前必须清零
- 关键命令回归集：100% 通过

## 6. 重点风险项测试
1. 多进程采样未完全 join 导致数据不完整
2. 参数解析覆盖（用户显式参数被默认逻辑覆盖）
3. memory 指标口径偏差（rusage/max_mem）
4. 不同 OS 下路径/权限/依赖差异
5. 大时间跨度查询性能退化

## 7. 测试环境矩阵
- 环境 A：CentOS 7.9（主验证）
- 环境 B：Rocky 8.10（兼容验证）
- Python：按项目目标版本执行（建议 3.8+ 兼容，按发布策略执行）
- LSF：提供真实集群样本 + mock 数据回放两套路径

## 8. GUI 测试执行策略（补充）
1. **最小自动化冒烟（每次提交执行）**
   - GUI 可启动
   - 主要页签可切换
   - 关键筛选可触发刷新
   - 基础导出可成功生成文件
2. **回归自动化（每日/每周）**
   - 关键筛选组合回归
   - 图表与表格数据一致性校验
3. **人工探索测试（每个里程碑）**
   - 可视化可读性与操作流畅度
   - 边界数据场景（空数据、超大时间范围）
   - 报表对外可用性（截图、导出内容）

## 9. 测试产出物
- 测试计划
- 测试用例集（含 GUI 专项用例）
- 每轮测试报告（通过率/缺陷分布/阻塞项）
- 发布前质量评估结论（Go/No-Go）
